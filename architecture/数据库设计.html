数据库设计

一、业务实体与功能对应

1.1 用户与权限模块

表名说明主要功能/场景
user用户基础信息（车主/管理员/运营方）注册、登录、实名认证、权限管理
role角色定义（车主/管理员/运营方/访客等）权限分级、菜单授权
user_role用户与角色关联表一用户多角色，灵活授权
permission权限/菜单资源控制 API/页面/功能访问
role_permission角色-权限关联表角色资源分配，细粒度管控

1.2 车辆管理与绑定

表名说明主要功能/场景
vehicle车辆基础档案与实时状态车辆注册、档案、状态监控
user_vehicle用户-车辆多对多绑定关系绑定/解绑、企业与个人共管车辆
vehicle_status车辆状态快照/历史轨迹定时采集、地图定位、历史分析
vehicle_picture车辆图片车证、照片、识别等

1.3 充电桩与服务

表名说明主要功能/场景
charging_pile充电桩信息（编号、位置、状态、品牌）地图找桩、桩状态展示、筛选
charge_record充电全过程记录扫码启动、状态推送、账单分析
charge_payment充电付费记录、退款、开票等支付、发票、对账

1.4 能耗与数据大屏

表名说明主要功能/场景
energy_data车辆能耗与统计数据能耗分析、排行、趋势
dashboard_stat大屏统计/可视化展示缓存数据ECharts 大屏、业务运营分析

1.5 告警与消息

表名说明主要功能/场景
alarm_record告警事件（车辆、充电、系统）故障/充电异常、预警通知
notification消息推送/消息中心用户/管理员推送、事件订阅
feedback用户反馈与建议用户投诉、建议、人工回复

1.6 日志与操作归档

表名说明主要功能/场景
operation_log用户操作/系统行为日志审计、归档、运维、数据回溯
login_log登录/登出/异常行为日志安全审计、风控、用户追踪

1.7 第三方/平台对接与扩展

表名说明主要功能/场景
third_party第三方平台接入信息充电桩/地图/支付等对接
third_callback第三方回调记录状态同步、对账、事务溯源

1.8 多租户/组织架构（可选扩展）

表名说明主要功能/场景
tenant平台多租户主表企业用户/分区数据隔离
org企业组织/部门架构表组织级运营、管理分权

二、关键字段设计与索引

以部分核心表为例，更多表结构可按上述模板展开：

2.1 用户表 user

字段类型约束说明
idBIGINTPK, AUTO_INCREMENT用户唯一 ID
usernameVARCHAR(32)UNIQUE, NOT NULL登录账号/手机号
passwordVARCHAR(128)NOT NULL加密密码
real_nameVARCHAR(32)真实姓名
roleVARCHAR(16)INDEX角色
avatarVARCHAR(128)头像 URL
statusTINYINT状态(0 禁用/1 正常)
tenant_idBIGINT多租户 ID(可选)
created_atDATETIME注册时间

建议索引：

  • username唯一索引
  • role, tenant_id, status联合索引

2.2 车辆表 vehicle

字段类型约束说明
idBIGINTPK, AUTO_INCREMENT车辆唯一 ID
plate_numberVARCHAR(20)UNIQUE, NOT NULL车牌
vinVARCHAR(32)UNIQUE车架号
brandVARCHAR(32)品牌
statusVARCHAR(16)状态
tenant_idBIGINT所属租户 ID（扩展）
create_timeDATETIME创建时间

索引建议：

  • plate_number唯一索引
  • tenant_id, status复合索引

2.3 用户-车辆绑定 user_vehicle

字段类型约束说明
idBIGINTPK主键
user_idBIGINTFK用户 ID
vehicle_idBIGINTFK车辆 ID
bind_timeDATETIME绑定时间

联合唯一索引：user_id + vehicle_id

2.4 充电桩 charging_pile

字段类型约束说明
idBIGINTPK桩唯一 ID
codeVARCHAR(32)UNIQUE桩编号
locationVARCHAR(128)地理位置
statusVARCHAR(16)状态
brandVARCHAR(32)品牌
powerDECIMAL(6,2)功率
create_timeDATETIME创建时间

2.5 充电记录 charge_record

字段类型约束说明
idBIGINTPK记录 ID
user_idBIGINTFK车主 ID
vehicle_idBIGINTFK车辆 ID
pile_idBIGINTFK桩 ID
start_timeDATETIME充电开始时间
end_timeDATETIME结束时间
energyDECIMAL(8,2)充电电量(KWh)
costDECIMAL(8,2)充电费用
statusVARCHAR(16)充电状态

2.6 告警表 alarm_record

字段类型约束说明
idBIGINTPK告警唯一 ID
user_idBIGINTFK用户 ID
vehicle_idBIGINTFK车辆 ID
typeVARCHAR(16)告警类型
descriptionVARCHAR(128)告警说明
create_timeDATETIME时间
levelTINYINT级别
statusTINYINT处理状态

其余表结构可按此规范自扩展。

四、数据一致性、冗余与隔离

  • 主外键强约束，所有外链加索引，防止脏数据、提升检索效率
  • 数据冗余最小化：仅核心业务数据做主表，聚合/分析用视图或缓存表
  • 多租户隔离：tenant_id 字段横贯所有业务主表
  • 操作日志、历史归档：定期归档，历史表与主表分离，支持追溯与性能分担
  • 字典表/枚举管理：车辆类型、告警类型等业务枚举均建独立字典表

五、数据初始化、测试、备份与安全

  • 提供一键建表/基础数据/Mock 数据 SQL 脚本
  • 支持开发、测试、生产多套数据库环境与自动脱敏
  • 每日自动备份，异地容灾，备份文件专人维护归档
  • 密码/手机号等敏感字段加密，接口全链路 HTTPS 传输
  • 数据权限基于用户/组织/租户三重校验，API 和 DB 中间件双层隔离

六、可扩展与归档建议

  • ER 图、表结构 SQL、数据字典、主外键关系文档全部归档
  • 重要结构/字段/索引变更均需有变更说明、测试记录与归档日志
  • 支持 AIGC/低代码平台一键生成模型/接口/文档
