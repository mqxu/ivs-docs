部署说明文档

一、部署环境要求

1.1 硬件与系统

组件最低配置要求推荐配置
服务器4 核 CPU、8G 内存、50G 磁盘8 核 CPU、16G 内存、200G 磁盘
操作系统Linux (CentOS 7+/Ubuntu 18+)Ubuntu 22.04 LTS/CentOS Stream
网络10Mbps 以上100Mbps/千兆专线

1.2 软件依赖版本

  • JDK：Java 17+（推荐 OpenJDK 17/21）
  • Node.js：16.x 及以上
  • Nginx：1.18+（前端静态资源、反向代理）
  • MySQL：8.0+
  • Redis：6.0+（缓存、消息队列）
  • Docker：20.10+（可选）
  • Kubernetes：1.23+（可选）

二、前端/后端/数据库/大屏服务部署流程

2.1 数据库（MySQL）

  1. 安装 MySQL 8.0+，新建数据库如 smart_vehicle
  2. 执行建表与初始化 SQL 脚本（见 /db/init.sql）
  3. 设置远程访问（如有多机部署），注意账号密码安全

2.2 后端（SpringBoot）

  1. 安装 JDK，拉取代码或上传构建包
  2. 配置application-prod.yml（数据库、Redis、接口、密钥等）
  3. 运行命令：java -jar smart-vehicle-server.jar --spring.profiles.active=prod
  4. 检查日志确保服务启动，常见端口8080（可通过 Nginx 反代）

2.3 前端（UniApp/Vue3）

  1. 安装 Node.js 与 pnpm/yarn
  2. 拉取前端代码，配置 .env.prod （API_BASE_URL 等）
  3. 编译打包：pnpm build 或 yarn build
  4. 将/dist目录部署至 Nginx 静态资源目录（如/usr/share/nginx/html）
  5. 配置 Nginx 代理前端接口到后端服务

2.4 大屏服务（ECharts 可视化）

  • 如大屏为独立服务：参考前端部署，/dashboard路由指向独立静态站或子应用
  • 若为微前端：注册至主应用框架，保证 API 与静态资源一致
  • 确保大屏与后端接口连通，地图/实时数据正常渲染

三、配置文件、环境变量与参数设置说明

3.1 后端配置

  • application-prod.yml示例（重点参数）：

    server:
  port: 8080
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_vehicle?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
    username: root
    password: 123456
  redis:
    host: 127.0.0.1
    port: 6379
jwt:
  secret: xxxxxxx
  expire: 86400

  • 环境变量优先级高于 yml 配置（如云主机可用SPRING_DATASOURCE_URL等环境注入）

3.2 前端配置

  • .env.prod 示例

    VITE_API_BASE_URL=https://api.smartvehicle.com
VITE_MAP_KEY=xxxxxx

四、Docker/K8s 容器化与自动化部署方案

4.1 Docker 单体部署

  • Dockerfile（后端示例）：

    FROM openjdk:17-jdk-alpine
COPY smart-vehicle-server.jar /app.jar
ENTRYPOINT ["java","-jar","/app.jar"]

  • 前端 Nginx 镜像：

    FROM nginx:1.22
COPY dist/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/nginx.conf

  • docker-compose.yml 样例（后端+前端+MySQL+Redis）：

    version: "3"
services:
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: smart_vehicle
    ports:
      - "3306:3306"
  redis:
    image: redis:6
    ports:
      - "6379:6379"
  backend:
    build: ./server
    ports:
      - "8080:8080"
    depends_on:
      - db
      - redis
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

4.2 Kubernetes 云原生部署（选配）

  • 提供后端、前端、数据库、Redis、Nginx 等 K8s yaml 示例
  • 推荐采用 Helm、CI/CD 平台（如 GitLab CI）自动化部署

五、部署常见问题与排查建议

  • 端口占用：检查netstat -anp | grep 8080/80/3306，修改配置文件端口或释放端口
  • 数据库连接失败：核查数据库主机、端口、账号权限、防火墙设置
  • 前端 404 或接口跨域：确认 Nginx 转发配置、API 地址、CORS 设置
  • 容器启动异常：用docker logs或kubectl logs检查启动日志
  • 内存/磁盘不足：定期清理日志与临时文件，检查服务器空间
  • 外部 API/地图不可用：核查 API KEY 与网络连通性

六、部署流程归档和演示材料要求

  • 每次部署均需归档完整部署脚本、配置文件、主要操作步骤与截图、部署/验收日志
  • Dockerfile、docker-compose、K8s yaml 等自动化配置文件单独归档
  • 推荐录制关键部署环节视频或 PPT，方便评审/教学/查阅
  • 归档材料统一存放于 /deploy/ 目录，版本同步管理
