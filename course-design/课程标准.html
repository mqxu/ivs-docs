课程标准

一、课程定位

本课程面向高职本科、应用型本科等院校“软件工程”“移动应用开发”等相关专业，是一门以赛项和企业项目为驱动，融合理论与实践、以就业能力与创新素养为核心的专业主干课程。课程紧贴智能车载与新一代信息技术产业发展趋势，围绕移动应用开发最新标准，兼顾行业实际、岗位需求与大赛导向，实现“课程-项目-赛项-就业”一体化设计。课程适用于希望掌握企业级项目开发流程、具备跨端开发与大数据可视化能力、参与创新竞赛或行业真实项目的高年级学生。

二、课程设计

课程采用“项目驱动+案例教学+竞赛一体化”模式，围绕“智能车载系统”完整业务流程，将项目开发流程细分为原型设计、客户端/服务端开发、数据管理、数据可视化、系统集成与创新答辩等多个模块。设计中充分融入移动应用开发大赛标准、企业真实场景、团队协作机制和 AIGC 新技术应用，保证内容体系完备、实训任务丰富、考核标准多元，能满足高阶能力培养和多元归档材料生成。

三、课程目标

能力目标

  1. 能独立完成智能车载系统项目的需求分析、功能设计与原型开发。
  2. 能熟练开发移动端应用，掌握 UniApp/Vue3 的多端适配与组件化编程。
  3. 能独立设计与实现后端业务系统，使用 SpringBoot 开发标准 RESTful API 并确保安全与扩展性。
  4. 能对接第三方服务（如地图 API、充电桩接口），实现多系统集成与数据联动。
  5. 能进行复杂业务数据库建模、SQL 优化及数据安全管理。
  6. 能使用 ECharts 进行多维数据可视化与大屏开发，支撑业务运营分析。
  7. 能编写自动化测试用例，完成 API 测试与系统集成，掌握持续交付与部署流程（CI/CD、Docker 等）。
  8. 能胜任赛项仿真与团队项目路演，具备现场答辩与问题攻坚能力。

知识目标

  1. 掌握智能车载行业应用场景、业务流程与主流技术架构。
  2. 理解项目驱动开发模式与企业级移动应用全流程开发规范。
  3. 熟悉前后端分离开发原理，掌握 Vue3、UniApp、SpringBoot 等主流技术原理及实践。
  4. 理解常见地图服务（如高德、腾讯 API）集成原理及位置数据处理方法。
  5. 掌握数据库建模、E-R 图绘制、主外键设计与常用 SQL 优化策略。
  6. 熟悉数据可视化基本方法与 ECharts 常用图表类型、组件联动机制。
  7. 了解接口测试、系统集成、持续集成/部署的标准流程与工具应用。
  8. 理解移动应用开发大赛评分标准及企业项目归档评优体系。

素质目标

  1. 培养团队协作精神与项目管理意识，能在多角色环境中高效分工与沟通。
  2. 强化工程伦理与数据安全意识，遵守技术开发与数据合规法规。
  3. 培养良好的学习习惯与自我驱动能力，善于利用新技术解决实际问题。
  4. 培养主动创新、持续改进与自我超越的职业精神。
  5. 强化代码规范、文档归档和过程复盘的职业素养。
  6. 提高分析问题、归纳总结与解决复杂工程问题的综合素质。
  7. 培养结果导向和质量意识，注重交付标准和项目材料归档完整性。
  8. 培养时间管理、抗压能力和现场表达展示能力。

思政目标

  1. 增强国家战略意识，理解智能网联汽车、新一代信息技术等领域对国家发展的重要意义。
  2. 培养创新精神和担当意识，激发为科技强国、智慧交通等行业发展贡献力量的理想信念。
  3. 培养诚信守法、合作共赢、敬业奉献的社会责任感和职业道德。
  4. 强调数据安全、用户隐私和绿色低碳等社会责任，树立可持续发展的意识。
  5. 引导学生树立“工匠精神”，追求卓越、精益求精、注重细节和品质。
  6. 弘扬团队协作、互帮互助、集体奋斗的优良传统，激励班级与团队凝聚力。
  7. 培养危机意识和风险管理能力，懂得在技术创新中防范风险、守住底线。
  8. 引导学生关注行业最新政策法规和社会热点问题，做到知行合一、服务社会。

四、课程内容与要求

课程内容结构化分为八大模块：

  1. 项目启动与需求分析（2 学时）：理解车载行业背景，梳理业务需求，完成用户故事和用例文档；
  2. 原型设计与评审（2 学时）：高保真原型输出、交互流程与评审材料归档；
  3. 客户端开发（UniApp/Vue3）（12 学时）：实现地图定位、扫码充电、用户管理、组件封装、前端联调；
  4. 服务端开发（SpringBoot）（10 学时）：开发 RESTful API、接口安全、后端业务逻辑与接口文档输出；
  5. 数据库设计与管理（MySQL）（4 学时）：业务数据建模、SQL 优化、数据备份和数据安全；
  6. 数据大屏与可视化（ECharts）（6 学时）：多维数据展示、实时大屏开发、业务分析与交互体验优化；
  7. API 测试与系统集成（6 学时）：接口自动化测试、Mock 联调、系统一键部署与集成；
  8. 赛项仿真与创新展示（2 学时）：仿真大赛流程、创新功能开发、团队答辩与路演演示。

每个模块均配阶段成果、归档要求与创新任务，学生需分组协作、全流程产出项目成果包。

五、课程教学安排

  • 总学时建议为 64 学时，16 周，每周 4 学时（见教学进度表）；
  • 课程以“讲授+实训+项目+答辩”混合式推进，前半学期聚焦技能训练与模块开发，后半学期聚焦系统集成、创新攻关与材料归档；
  • 重要节点包括需求分析评审、原型展示、阶段测试、系统演示、答辩创新与成果归档，每阶段均配材料包与过程记录。

六、课程资源与支撑条件

  • 硬件资源：可用机房、移动设备、服务器、网络环境等；
  • 软件/平台：HBuilderX、VSCode、Navicat、Postman、Docker、Jenkins、Git、Axure/Figma、ECharts、MySQL、SpringBoot、UniApp/Vue3 等全链条工具和开发环境；
  • 课程材料：案例库、视频库、标准化文档模板、示例代码、错题集、评分细则、材料归档手册等；
  • 校企合作/赛项资源：行业专家讲座、赛项历年真题、企业导师、仿真评审机制。

七、教学方法

采用“项目驱动+分层教学+团队协作+案例引领+竞赛仿真”多元融合教学。教师通过阶段任务包、典型案例、答辩演练、难点攻关、同伴互评和创新挑战，激发学生主动学习和分层突破。引入 AIGC 自动化工具赋能作业批改、材料归档和项目过程管理，促进因材施教和教学质量提升。

八、教学评价

  • 实行“过程评价+结果评价+创新加分”三维评价体系，过程性评价侧重过程日志、阶段成果、协作记录，结果性评价侧重功能实现、系统集成、演示答辩，创新评价关注新技术应用、难点突破和项目亮点。
  • 所有评价均需有材料支撑，配标准评分表、评优案例和自评/互评表，所有材料归档可查。
  • 教学评价机制对接赛项评分体系，实现课程与大赛评价双通道。

九、其他说明

课程实施过程中，强调“持续迭代、过程归档、创新驱动、团队协作”，鼓励教师根据行业与赛项最新要求不断更新教学内容、案例与评价体系。课程成果可对接毕业设计、创新创业、企业实训与赛项申报，力求“一课多用、成果可迁移、过程可追溯、材料可归档”。
