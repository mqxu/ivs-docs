二、接口参数、类型与结构示例

1. 用户登录

  • URL：POST /api/user/login

  • 请求参数：

    参数名类型必填说明
    usernamestring是手机号/账号
    passwordstring是密码

  • 响应参数：

    字段类型说明
    codeint状态码（0=成功）
    msgstring消息
    data.tokenstringJWT Token
    data.userobject用户基本信息

  • 返回示例：

    {
  "code": 0,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6...",
    "user": {
      "id": 10001,
      "username": "testuser",
      "role": "car_owner"
    }
  }
}

2. 查询车辆列表

  • URL：GET /api/vehicle/list

  • 请求参数：

    参数名类型必填说明
    pageint否分页页码
    pageSizeint否每页数量

  • 响应参数：

    字段类型说明
    codeint状态码
    msgstring消息
    data.listarray车辆列表
    data.totalint总条数
    data.list[].idint车辆 ID
    data.list[].plate_numberstring车牌
    data.list[].brandstring品牌
    data.list[].statusstring状态

  • 返回示例：

    {
  "code": 0,
  "msg": "success",
  "data": {
    "list": [
      {
        "id": 501,
        "plate_number": "粤A12345",
        "brand": "BYD",
        "status": "正常"
      }
    ],
    "total": 1
  }
}

3. 启动充电

  • URL：POST /api/charge/start

  • 请求参数：

    参数名类型必填说明
    vehicle_idint是车辆 ID
    pile_idint是充电桩 ID

  • 响应参数：

    字段类型说明
    codeint状态码
    msgstring消息
    dataobject启动充电详情

三、认证/鉴权机制说明

  • JWT（JSON Web Token）：用户登录后获得 token，所有需认证接口需在请求头加Authorization: Bearer {token}。
  • 接口权限：不同角色（如车主、管理员、运营方）访问不同接口范围，后端基于 token 验证和角色判断。
  • 刷新机制：Token 过期需重新登录获取；支持刷新 token 的接口建议实现。

四、典型接口调用流程

登录-获取 token-访问受保护资源

sequenceDiagram
    participant 前端
    participant API
    前端->>API: POST /api/user/login (账号/密码)
    API-->>前端: 返回token
    前端->>API: GET /api/vehicle/list (带token)
    API-->>前端: 返回车辆数据
  • 建议：前端拦截器自动携带 token；token 失效需自动跳转登录页并提示。

五、状态码及异常返回说明

状态码含义说明
0成功
401未认证/Token 失效需重新登录
403权限不足仅授权用户可操作
404资源不存在
500服务器异常请联系管理员
1001参数错误参数校验失败
1002业务规则冲突绑定、充电、解绑等异常

  • 异常返回示例：

    {
  "code": 401,
  "msg": "认证失败，请重新登录"
}

六、版本管理与文档归档要求

  • 版本控制：API 采用 URI 或 Header 区分版本（如/api/v1/user/login），每次 API 升级需标注版本、变更说明。
  • 文档归档：
    • 每次迭代/评审/验收节点，需归档最新接口文档（如 Swagger/OpenAPI 导出 JSON、HTML、PDF 等格式）。
    • 需同步维护接口变更日志、兼容性说明与测试用例表。
    • 推荐定期 API 文档评审，归档在企业文档系统、Notion、团队云盘等。

七、推荐 API 文档工具

  • Swagger/OpenAPI：自动生成、前后端同步、可在线调试、JSON/YAML/HTML/PDF 一键导出。
  • Apifox：接口管理+测试+Mock+文档一体化，支持团队协作、断言、数据流图。
  • Postman/ApiPost：接口调试、自动化测试脚本。
  • Rap2/YApi：国产开源 API 协作与 Mock 平台，适合校企团队。
