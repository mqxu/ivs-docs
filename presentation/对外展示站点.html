对外展示站点设计文档

一、项目首页

  • 内容建议：

    • 项目 LOGO、宣传 Slogan、智能出行愿景
    • 简明产品介绍（智能车载平台、多端协同、绿色出行赋能）
    • 核心价值点罗列：一站式车辆管理、智能充电、数据可视化、安全告警
    • 行业趋势与用户痛点背景

  • 页面样例：

    | LOGO | 智能车载系统——赋能智慧出行新生态 |
| 产品简介 | 核心功能 | 行业背景 | 用户价值 |

二、功能演示页面

  • 内容建议：

    • 交互式 Demo：如车辆绑定、扫码充电、数据大屏
    • 视频/动画：实际操作流畅演示、典型应用场景短视频
    • 体验区：模拟登录/车辆管理/地图找桩等交互

  • 技术提示：

    • 推荐嵌入动态演示（如 Vue/React 组件、iframe、视频播放）
    • 视频支持自动播放、弹窗讲解

三、技术架构与技术栈介绍

  • 内容建议：

    • 架构总览图（Mermaid/PlantUML/ECharts 可视化）
    • 技术栈详细说明：UniApp/Vue3（前端）、SpringBoot（后端）、MySQL/Redis、ECharts（可视化）
    • 创新点与行业主流对比（如多端适配、实时推送、微服务等）

  • 页面样例：

    graph LR
  App/小程序 --API/WS--> 后端
  后端 --SQL--> MySQL
  后端 --API--> 可视化大屏

四、团队与项目进展

  • 内容建议：

    • 团队成员简介、分工、风采展示
    • 主要项目节点与里程碑（甘特图/时间轴）
    • 近期进展、获奖或应用落地成果展示

  • 技术提示：

    • 推荐采用时间轴/进度条可视化展示
    • 支持团队照片/个人介绍模块

五、资料下载与 API/文档开放入口

  • 内容建议：

    • 产品手册、用户操作指南、课程归档包
    • API 文档入口（Swagger/OpenAPI/Apifox 嵌入）
    • 下载区：Demo App/安装包、数据报表、项目 PPT 等
    • 开放接口申请与测试账号指引

  • 技术提示：

    • 支持静态文件一键下载，API 文档推荐用 Swagger UI 在线预览

六、展示站点部署与运维建议

  • 部署平台推荐：

    • 静态站：Vercel、Netlify、GitHub Pages、阿里云 OSS
    • 动态站/SSR：Vercel（Next.js）、阿里云 ECS、服务器自托管

  • 自动化建议：

    • 持续集成（如 GitHub Actions 自动构建、发布）
    • CDN 全网加速，HTTPS 全站加密
    • 日志监控与站点健康检查

  • 归档建议：

    • 展示站源码、部署脚本、配置文件归档于 /website/ 目录
    • 每次大改动留版本与变更说明

七、对外联系与交流方式

  • 官方邮箱、技术支持微信、企业微信二维码
  • 站内留言/反馈入口
  • 开源仓库/合作伙伴链接
  • 在线客服/FAQ 文档
