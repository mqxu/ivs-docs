页面结构与交互说明

一、核心页面模块/组件结构

1. 首页（Dashboard）

  • 顶部导航栏：系统标题、消息入口、用户头像
  • 快捷入口区：如“扫码充电”、“查找充电桩”、“历史数据”
  • 数据总览卡片：车辆状态、剩余电量、告警统计
  • 轮播公告区：运营通知、系统更新提示
  • 底部 Tab 栏：主页、车辆管理、地图充电、数据中心、个人中心

2. 车辆管理页

  • 车辆列表区：每辆车卡片、状态、解绑按钮
  • 添加车辆入口：浮动按钮
  • 车辆详情弹窗：车辆信息、历史数据、解绑操作

3. 充电地图页

  • 地图展示区：实时显示当前位置与附近充电桩
  • 充电桩标记点：点击弹出桩详情卡片
  • 筛选/排序按钮：按空闲、距离、品牌筛选
  • 导航与扫码入口：一键跳转至导航/扫码充电

4. 历史数据页

  • 筛选栏：车辆、日期区间选择
  • 数据卡片区：充电、里程、能耗等趋势统计
  • 导出按钮：导出 PDF/Excel 报表

5. 个人中心/设置页

  • 用户资料：昵称、手机号、实名认证状态
  • 账户与权限管理
  • 系统设置、帮助与反馈

二、页面导航与跳转逻辑

  • 底部 Tab 栏：全局主导航，随时切换主模块，状态记忆（如返回上次子页）。
  • 页面内跳转：如车辆卡片点进详情，支持“返回”按钮返回上一页。
  • 抽屉/侧边栏：用于二级菜单或批量操作（如多车管理、批量导入）。
  • 弹窗/浮层：重要操作如解绑确认、充电进度、错误提示均采用弹窗。
  • 页面栈管理：保证操作流程中多级页面支持后退，弹窗自动聚焦最新操作项。
  • 返回机制：物理/虚拟“返回”键均优先关闭弹窗或浮层，再回退页面。

三、主要业务流程交互步骤

1. 数据查询流程

  1. 用户进入历史数据页，选择车辆和日期区间
  2. 系统加载数据（加载动画显示），展示统计卡片与趋势图
  3. 用户可点击导出按钮，一键导出数据报表

2. 车辆解绑流程

  1. 用户在车辆管理页点击目标车辆的“解绑”按钮
  2. 弹窗二次确认（含车辆信息与风险提示）
  3. 用户确认解绑，系统请求后端接口
  4. 成功后刷新列表并反馈操作成功（toast 或弹窗）

3. 充电流程

  1. 用户在地图页选择充电桩，点击“扫码充电”
  2. 系统打开摄像头扫码，识别成功后跳转充电详情页
  3. 实时显示充电状态、剩余时间、停止充电按钮
  4. 充电完成自动提示，用户可评分或查看历史

四、典型交互场景伪代码与流程图

1. 伪代码（以扫码充电为例）

// 用户扫码充电主流程
onScanQRCode() {
  openCamera();
  if (scanSuccess) {
    showLoading('正在连接充电桩...');
    post('/api/charge/start', { code: qrCode }).then(res => {
      if (res.success) {
        navigateTo('ChargeDetail', { recordId: res.data });
      } else {
        showError(res.message);
      }
    });
  } else {
    showError('二维码无效或扫码失败');
  }
}

五、辅助交互设计要素

  • 加载动画：所有耗时操作（如数据加载、扫码连接、报表导出）均有统一加载动画，动画时间 ≥0.3s，最大 3s 自动超时反馈。
  • 提示与引导：常见操作配气泡提示/引导页，首次进入新功能自动弹窗说明。
  • 错误处理：所有失败场景均给出明确错误信息及修正建议（如网络异常、接口失败、权限不足），Toast 与弹窗结合。
  • 状态反馈：所有操作有同步/异步状态反馈（如按钮变 loading、进度条、完成弹窗/Toast）。
  • 输入校验：表单输入校验，异常高亮，禁止非法输入/空提交。

六、移动端操作友好性建议

  • 点击区域充足：核心按钮和可交互区 ≥44x44px，防止误触。
  • 滑动与手势优化：地图/列表页支持惯性滑动、下拉刷新、左右滑动删除。
  • 键盘适配：表单输入时自动弹出合适键盘类型（如数字/日期），并自动收起。
  • 短路径设计：关键功能不超过 3 步直达，减少层级跳转。
  • 自适应适配：界面在不同屏幕（大屏、小屏、Pad）下无错位、核心功能无隐藏。
  • 节电与性能：地图、动画等页面自动降频或省电模式，低性能设备下关闭高耗能动效。
