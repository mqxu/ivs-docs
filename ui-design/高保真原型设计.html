高保真原型设计

一、核心页面与功能区原型

1. 主要原型页面列表

  • 首页：系统总览、快捷入口、消息通知与状态提示。
  • 车辆管理页：车辆列表、详情、状态展示、车辆绑定/解绑操作入口。
  • 充电地图页：地图展示、充电桩分布、导航入口、空闲状态显示、桩点筛选。
  • 扫码充电页：扫码入口、充电状态流程、实时进度、充电完成提示。
  • 历史数据页：能耗统计、行驶轨迹、充电记录、数据导出。
  • 个人中心/设置：用户资料、认证、消息订阅、帮助与反馈。

2. 核心功能区高保真原型设计要点

  • 每页核心功能区采用卡片、列表、图表、地图等多种视觉模块清晰布局。
  • 常用操作放置于显眼位置，状态与告警消息采用弹窗/红点等视觉高亮。
  • 所有页面应遵循行业通用 UI 设计规范（如 Material Design、Ant Design Mobile 等）。

交付标准与落地要点

  • 每页原型需配详细交互注释和功能区说明，支持 AIGC 自动识别与材料归档。

二、交互流程与主用例场景

1. 主线交互流程梳理

  • 车辆添加/绑定流程：用户进入车辆管理页 → 点击“添加车辆” → 填写车辆信息/扫码识别 → 完成绑定，系统提示成功。
  • 充电流程：用户地图查找桩点 → 选定目标 → 扫码进入充电 → 实时显示进度 → 充电完成弹窗提醒。
  • 历史数据查询：首页快捷入口/菜单 → 进入历史数据页 → 按日期/车辆筛选查询 → 查看统计报表 → 支持导出 PDF/Excel。

2. 用例交互流程图（Mermaid 示例）

交付标准与落地要点

  • 每条主用例交互均须配详细原型流程，归档至用例/需求章节，便于后期开发与评审查阅。

三、响应式布局与多端适配

1. 设计规范

  • 所有原型页面支持移动端（主）、小程序、Pad等多端适配，采用响应式布局思想。
  • 元素自适应尺寸，地图、卡片、表格等主要区域在不同设备屏幕下保持核心功能不变形、不遮挡。
  • 推荐采用 12 栅格系统、弹性盒（Flex）布局、断点自适应等主流响应式设计技术。

2. 适配效果示意（Figma/Axure 等交互注释）

  • 同一功能区分别展示在iPhone SE、iPad、Android Pad、主流小程序端的版面布局。
  • 关键按钮、输入框、图表在小屏和大屏下自动伸缩、对齐、分栏或折叠。

交付标准与落地要点

  • 各端适配原型页面需逐版留存，配设备适配说明与注释，归档原型材料，便于评优复查。

四、典型操作流程原型

1. 示例 1：扫码充电流程原型

  • 步骤 1：地图页点击充电桩 → 弹出“扫码充电”入口
  • 步骤 2：调取扫码摄像头 → 扫码后显示充电详情页
  • 步骤 3：充电状态实时展示（进度条/剩余时间/费用/终止按钮）
  • 步骤 4：充电完成 → 弹窗提示并引导评分/分享

2. 示例 2：历史轨迹查询流程原型

  • 步骤 1：首页点击“历史数据”→ 进入数据筛选页面
  • 步骤 2：选择车辆、日期区间 → 地图展示轨迹与数据统计
  • 步骤 3：可导出行驶轨迹报告 PDF/Excel

3. 示例 3：告警提示与异常流程

  • 实时告警以红色浮窗、状态栏提醒等高亮形式呈现，点击可查看详细说明并支持一键反馈/报修。

交付标准与落地要点

  • 所有关键流程原型需配注释和操作说明，支持开发与测试团队快速对照实现，归档至流程材料库。

五、推荐工具与交付标准

1. 推荐工具

  • Axure RP：专业级高保真原型设计与交互流程模拟，支持分阶段版本管理与团队协作。
  • Figma：在线协作、多人实时编辑、响应式适配强，适合移动端、Web、小程序一稿多用。
  • 墨刀（MockingBot）：国产协同原型工具，简易上手、便于快速迭代，适合教学与初创团队。
  • （如有深度开发团队可结合 Sketch、Adobe XD 等主流设计工具）

2. 原型交付标准

  • 每次评审/阶段性开发节点须归档当前版本高保真原型文件（.rp/.fig/.xd 等格式）与主要页面 PNG/JPG 截图。
  • 原型需配交互注释、功能说明、页面流程图，并以“版本+日期”编号归档。
  • 归档包应含“设备适配说明”、“页面交互流程”、“功能区说明”等文档。

交付标准与落地要点

  • 所有原型设计文件须支持一键打包、历史版本回溯、团队协同编辑和阶段评优申报。
  • 建议建立原型材料专属目录与归档规范，便于查阅、评优与赛项申报。

六、原型文件归档建议

  • 原型文件按需求评审、迭代开发、功能上线等阶段分阶段归档，配“阶段标签”与版本号；
  • 所有主要页面/流程均需有高保真原型文件、交互流程和主要截图归档；
  • 建议每个版本配变更说明，归档原型库建议用 Notion、企业云盘或 Git LFS 等方式管理；
  • 归档清单需配原型目录、维护人、修改时间，方便团队追溯和 AIGC 后续自动化调用。
