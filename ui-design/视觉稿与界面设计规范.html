视觉稿与界面设计规范

一、品牌色、主辅色、字体、图标集规范

1. 品牌与主辅色

  • 品牌主色：#1B5CB2（深科技蓝，体现智能与可靠）

  • 辅色 1：#4AD0E7（科技青，强调创新和活力）

  • 辅色 2：#FFC93C（金橙，突出交互/警告/高亮）

  • 中性色：#F5F7FA（页面底色）、#E2E8F0（卡片背景）、#222B45（正文深色）

  • 配色示意：

    用途颜色值说明
    主色#1B5CB2标题、按钮主色
    辅色 1#4AD0E7图表、数据高亮
    辅色 2#FFC93C警告、操作提示
    中性/辅助色#F5F7FA背景、分割线

2. 字体规范

  • 中文：优先采用“思源黑体”、“PingFang SC”、“微软雅黑”；
  • 英文：Roboto、Helvetica Neue；
  • 正文：16px，标题 18~22px，导航/按钮 14px。

3. 图标集

  • 统一采用线性、简洁风格，推荐使用 iconfont 定制、Element Plus/Ant Design 图标集；
  • 主要功能区如地图、充电、车辆、告警、用户、设置等需专门设计高辨识度图标；
  • 图标大小规范：24x24px（主功能），16x16px（内嵌）。

交付标准与落地要点

  • 提供主色/辅色色卡 SVG、字体样例、全量 SVG 图标包，归档于“视觉规范材料库”。

二、组件/页面设计风格

  • 整体风格扁平化+简洁科技感，色块分明，留白充足，突出信息层级与交互友好；
  • 页面布局采用卡片式、分区明确、数据和功能入口一目了然；
  • 常用操作（如充电、定位、扫码）采用大按钮、色彩高亮、动效提示；
  • 科技感增强建议：弱化阴影、微动画点缀、动态图表、地图动效等。

组件设计例

  • 表单输入框、主按钮、卡片区块、数据图表、弹窗等均有统一圆角（8px）、边距（16px）、阴影（小于 2px）；
  • 状态类组件（成功/警告/异常）采用品牌辅色背景与 icon 标识。

交付标准与落地要点

  • 每个主要页面/组件需配设计稿、交互注释，所有样式变动须有设计说明文档。

三、常用控件与组件库

  • 前端推荐：UniApp 自带组件、uView Plus、Element Plus（管理后台）、Ant Design Mobile；
  • 图表控件：ECharts；
  • 地图：uniapp 内置地图组件/腾讯地图/高德 JS API；
  • 表单、输入、弹窗、导航、列表等：优先选用 uView、Element Plus、Ant Design Mobile 等高一致性方案，确保主题色和交互风格统一。

交付标准与落地要点

  • 指定每类组件库版本与自定义样式扩展规范，归档组件库清单和用例代码。

四、响应式与无障碍设计要点

  • 响应式：采用 flex、grid 布局，12 栅格系统，适配主流移动设备、小程序和 Pad 端，核心功能在不同分辨率下自适应。
  • 关键按钮和主信息区始终居中、缩放或重排，重要功能优先展示，次要信息可折叠/抽屉式处理。
  • 无障碍：所有图片、按钮需有 alt 文本/ARIA 标签，色彩对比度 ≥4.5:1，重要操作区配键盘导航和语音辅助提示。

交付标准与落地要点

  • 归档各端适配截图、无障碍测试报告和主流设备兼容性清单。

五、交互动画与过渡效果

  • 动画风格：简洁流畅、反馈明确，杜绝炫技和花哨动画；
  • 主要按钮、卡片、弹窗采用淡入淡出、缩放、滑动等微动效；
  • 地图、数据图表切换带渐变、数据上升/下降动效；
  • 操作完成、异常提示有明确的视觉反馈（如震动/变色/弹窗）。

交付标准与落地要点

  • 交互动画须有交互流程演示 GIF/视频，动画方案与技术实现文档归档。

六、界面一致性、易用性与可维护性要求

  • 全局采用统一设计语言和组件规范，所有模块遵循同一品牌色、圆角、字体和交互逻辑；
  • 表单校验、反馈提示、状态切换等保持一致交互习惯；
  • 易用性优先，常用功能一键直达、输入最简化、交互路径最短；
  • 样式与交互均应通过 SCSS/LESS 等预处理器规范统一，方便主题定制和后期维护。

交付标准与落地要点

  • 归档全局样式表、组件复用规范、维护文档，配备界面一致性自测清单。

七、视觉稿交付与归档标准

  • 每一轮设计/迭代都需归档 Figma/Axure/Sketch 等源文件及主要页面 PNG/JPG 高清图；
  • 配色方案、字体、图标、动效、主要 UI 组件均需归档 SVGO/PNG/SVG 格式及用法说明；
  • 视觉稿交付包须包含：目录索引、页面分组、版本说明、适配说明、设计变更记录；
  • 所有材料归档在团队云盘/企业文档/Notion 等平台，方便追溯与查阅；
  • 视觉稿需支持 AIGC 自动识别与代码生成场景（如配设计 token/样式变量等）。
